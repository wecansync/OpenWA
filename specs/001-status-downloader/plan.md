# Implementation Plan: WhatsApp Status Downloader

**Branch**: `001-status-downloader` | **Date**: 2026-05-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-status-downloader/spec.md`

---

## Summary

Replace stub `getContactStatuses()` / `getContactStatus()` engine implementations with
real `getBroadcasts()` / `getBroadcastById()` calls from whatsapp-web.js. Add a new
`downloadStatusMedia()` engine method that calls `message.downloadMedia()` and streams
the result as a binary HTTP response. Add a Status page to the React dashboard with a
contact list and per-contact media grid with download buttons.

---

## Technical Context

**Language/Version**: TypeScript 5.x / Node.js 22 LTS

**Primary Dependencies**: NestJS 11.x (backend), React 18 + Vite (dashboard),
whatsapp-web.js 1.34.7 (already installed — no new dependencies needed)

**Storage**: None — all status data is ephemeral, fetched live from WhatsApp engine

**Testing**: jest (unit), no new tests required per spec

**Target Platform**: Linux server (self-hosted)

**Project Type**: Web service (NestJS API) + Web application (React dashboard)

**Performance Goals**: Status list loads in <5 seconds for up to 200 contacts

**Constraints**: No new npm packages; no DB schema changes; no breaking changes to
existing status endpoints

**Scale/Scope**: Single-instance, per-session, on-demand
